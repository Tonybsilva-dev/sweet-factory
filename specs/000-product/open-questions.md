# Perguntas em aberto

## Produto e negócio

1. O sistema será usado por apenas uma loja ou deve nascer preparado para multiempresa?
2. Haverá mais de um usuário acessando a aplicação?
3. Precisa de autenticação no MVP?
4. O usuário quer registrar venda manualmente ou importar de algum lugar?
5. Vai existir controle de caixa ou apenas vendas?
6. As vendas terão taxa de delivery?
7. As vendas terão taxa de marketplace?
8. Haverá produção por encomenda?
9. O cardápio diário será apenas interno ou público?
10. O cardápio diário precisa gerar link público?

## Estoque

1. A baixa de estoque será automática ao vender?
2. O estoque pode ficar negativo?
3. Haverá controle de validade?
4. Haverá controle por lote?
5. O sistema deve alertar ingrediente acabando?

## Precificação

1. A margem padrão será global ou por produto?
2. O arredondamento do preço sugerido deve seguir alguma regra?
3. O sistema deve sugerir preço mínimo?
4. O sistema deve permitir promoções?
5. Haverá custo fixo rateado, como energia, gás e aluguel?

## Dashboard

1. O dashboard deve considerar receita bruta ou líquida?
2. Descontos devem reduzir receita no dashboard?
3. Produtos cancelados entram no dashboard?
4. Gastos devem ser considerados pela data da compra ou pelo consumo estimado?

## Técnico

1. Qual framework será usado no backend?
2. Qual framework será usado no frontend?
3. O deploy será em VPS, Vercel, Render, Railway ou outro?
4. Será usado Mongoose, Prisma MongoDB ou driver nativo?
5. O projeto terá testes e2e?
